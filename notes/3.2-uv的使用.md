安装方式：使用 uv tool install mcp-server-fetch 成功安装了MCP服务器
运行方式：使用 uvx mcp-server-fetch 启动MCP服务器
问题所在：uvx 命令根本不看环境变量，直接尝试重新解析依赖并连接 PyPI，由于网络超时导致连接失败

将运行命令从 uvx 改为 uv tool run

修改位置：backend/ai_agent/mcp/mcp_manager.py

修改内容：

# 修改前
if command == "uvx" and args:
    tool_dir = str(Path(settings.MCP_SERVERS_DIR) / server_id)
    env = server_config.get("env", {})
    env["UV_TOOL_DIR"] = tool_dir
    config["env"] = env

# 修改后
if command == "uvx":
    config["command"] = "uv"
    config["args"] = ["tool", "run"] + args

为什么 uv tool run 比 uvx 更好？
uv tool run 是 uvx 的完整形式，对已安装工具的处理更稳定
uv tool run 能够正确使用已安装的工具，不需要重新解析依赖
避免了网络连接问题，启动速度更快
