@with_graph_builder 装饰器详解
核心功能
这个装饰器的作用是自动构建 LangGraph 图实例，并将编译好的图传递给被装饰的函数。它负责：

从配置获取当前模式（outline/creative 等）
导入并绑定工具
创建模型实例（支持多提供商）
构建系统提示词
创建 SQLite checkpointer 和 store
编译图并传递给函数


```py
@with_graph_builder
async def process_rollback(graph):
    """处理回档操作"""
    config: RunnableConfig = {"configurable": {"thread_id": thread_id}}
    
    # 使用 graph 进行操作
    states = []
    async for state in graph.aget_state_history(config):
        states.append(state)
    
    return {"result": "数据"}

# 调用时需要用 async for 遍历收集（装饰器将结果包装成生成器）
result = None
async for item in process_rollback():
    result = item

# 返回
return result

```

异步生成器函数
```py
@with_graph_builder
async def generate_summary(graph):
    """处理总结对话并返回流式响应"""
    config: RunnableConfig = {"configurable": {"thread_id": thread_id}}
    
    summarize_message = HumanMessage(content="@summarize")
    
    # 流式处理
    async for message_chunk, metadata in graph.astream({"messages": [summarize_message]}, config, stream_mode="messages"):
        if message_chunk.content:
            print(f"{message_chunk.content}", end="|", flush=True)
        
        serialized_chunk = message_chunk.model_dump()
        yield json.dumps(serialized_chunk, ensure_ascii=False) + "\n"

# 直接返回 StreamingResponse
return StreamingResponse(generate_summary(), media_type="text/event-stream")
```


关键要点
函数签名：被装饰的函数第一个参数必须是 graph（编译后的 LangGraph 图实例）
返回值处理：
如果是普通函数，装饰器会将返回值包装成单元素生成器
如果是生成器函数，装饰器会透传 yield 的内容
调用方式：
普通函数：需要用 async for 遍历获取结果
生成器函数：直接作为异步生成器使用（如 StreamingResponse）

图的方法：可以使用 graph.aupdate_state()、graph.astream()、graph.aget_state_history() 等异步方法操作图状态
