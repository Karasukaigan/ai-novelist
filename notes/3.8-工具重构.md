# 对langchain的@tool理解：

## 自定义工具名：
默认情况下,工具名称来自函数名。当你需要一些更具描述性的东西时,就把它推上去
@tool("web_search")  # Custom name

那我能不能直接中文命名？

## 自定义工具描述
覆盖自动生成的工具描述,以更清晰地提供模型指导
@tool("calculator", description="Performs arithmetic calculations. Use this for any math problems.")

这个倒是和写函数文档没区别，会自动提取的。

## 自定义arg的结构
使用 Pydantic 模型或 JSON 模式定义复杂输入:

from pydantic import BaseModel, Field
from typing import Literal

class WeatherInput(BaseModel):
    """Input for weather queries."""
    location: str = Field(description="City name or coordinates")
    units: Literal["celsius", "fahrenheit"] = Field(
        default="celsius",
        description="Temperature unit preference"
    )
    include_forecast: bool = Field(
        default=False,
        description="Include 5-day forecast"
    )

@tool(args_schema=WeatherInput)

对喽，可以在这个地方作文章

定义的模型，文档可以保留，精简一点就好，比如："""输入参数"""
然后将所有文档里的使用示例，全部放入数据模型的description，只保留最需要注意的部分，这样更简洁。

然后比较有意思的就是工具的高级用法了，或许好好利用，可以大幅减少工具无效上下文占用
比如，如果重点在一个文件，用工具获取该文件的“ref引用”，直接将引用添加到系统提示词
后续ai对这个文件的任何修改，引用都会直接更新，确保ai不用反复读取文件
开启引用模式，应该要同时控制其他工具的返回结果（只能是“文件操作成功”，最大程度减少上下文占用），甚至修改ai的工具调用信息（比如直接打上省略号）
这样应该可以减少90％的开销？
